# FarmFlow - Claude Code作業指示書

## 🎯 プロジェクト方針（必読）
**転職ポートフォリオ用MVP**: 完璧より動作する状態を優先
**期限**: 2025年8月完成 → 農業現場の実務経験をアピール
**技術**: Flutter + Riverpod + Firebase

## 📋 実装範囲（厳守）
### ✅ やること（4画面のみ）
1. **機械リスト画面** - トラクター一覧（メイン画面）
2. **機械詳細画面** - メンテナンス履歴表示
3. **機械追加画面** - 新規登録フォーム
4. **点検記録画面** - 始業前点検入力

### ❌ やらないこと（範囲外）
- コンバイン・田植え機
- 肥料・農薬管理
- 複数ユーザー管理
- 過度な機能分割

## 🗂 フォルダ構成（標準Flutter）
```
lib/
├── models/        # データモデル
├── providers/     # Riverpod状態管理
├── screens/       # 画面4つ
├── widgets/       # 再利用ウィジェット
└── data/         # ダミーデータ
```

## 🎨 デザイン指針
### 色使い
- **メイン**: 農業らしい緑系（#4CAF50）
- **アクセント**: ブルー系グラデーション
- **ステータス**: 緑（良好）/ オレンジ（要確認）/ 赤（要修理）

### UI方針
- **モダン**: マテリアルデザイン3
- **カード**: 角丸・影・適度な余白
- **アイコン**: Material Icons使用
- **直感的**: 色+アイコンでステータス表現

## ⚡ 作業時の指示

```

2. 過度な装飾は避ける

```

### エラー対応
```
1. 最小限の修正で解決

3. MVPの範囲を超えない
```
